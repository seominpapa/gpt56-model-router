# GPT-5.6 Model Router

프로젝트를 시작할 때 필요한 질문을 짧게 묻고, GPT-5.6 **Sol**, **Terra**, **Luna**와 적절한 추론 강도(`low`–`max`, `ultra`)를 추천하는 Agent Skill입니다. 추천 이유, 시간·토큰·비용의 범위 예측, 하위 에이전트 구성, 오류·재작업 기반 조정 정책, 적용 가능한 방법을 함께 제시합니다.

## 무엇을 해 주나요?

- 작업 난이도, 실패 비용, 속도, 반복량, 도구·병렬 작업 필요성을 간단히 파악합니다.
- 일반 업무는 `Terra + medium`을 기본으로 삼고, 저위험 대량 작업은 Luna로 낮추며, 복잡·고위험 작업은 Sol로 올립니다.
- `ultra` 또는 multi-agent는 독립적인 하위 과제로 나뉘는 큰 작업에만 권장합니다.
- 추천마다 난이도·실패 비용·속도·반복량에 근거한 이유와, 가정을 표시한 완료 시간·총 토큰·비용 범위를 제공합니다.
- 하위 에이전트를 제안할 때는 역할별 모델·추론강도·추천 이유, 프로젝트 총괄 단독 수행 대비 시간·토큰·비용 영향을 함께 비교합니다.
- 초기 추천에는 시간절감안·토큰절감안을 포함하지 않습니다. 반복 오류·재작업 신호나 사용자의 조정 요청이 있을 때만 모델·추론강도 조정안과 영향을 제시합니다.
- 기준 추천안을 제시한 뒤, 프로젝트 전체에 적용할 조정 정책을 한 번만 선택하도록 안내합니다.
- 사용자가 시작을 승인하면 프로젝트의 `.codex/agents/`에 고정 설정 파일을 만들고, 가능한 Codex 환경에서는 권장 설정의 **프로젝트 총괄 에이전트**를 먼저 생성합니다. 총괄은 작업을 직접 수행하거나, 독립 작업이 있을 때만 하위 에이전트를 구성합니다.

## 추천 결과에 포함되는 내용

각 추천은 다음 형식으로 읽기 쉽게 정리합니다.

```text
추천: Sol + high
이유: 복잡한 여러 소스를 검증해야 하고, 잘못된 결론의 재작업 비용이 크기 때문입니다.
예상: 6~10시간 · 총 1.2M~2.0M 토큰 · 비용은 현재 단가 확인 후 산정
프로젝트 총괄: Sol + high — 전체 작업, 통합, 최종 검증을 책임
하위 에이전트: 리서치(Terra + high), 검증(Sol + high), 통합(Sol + high)
  - 이유: 세 흐름이 독립적이므로 완료 시간은 줄일 수 있습니다.
  - 영향: 4~7시간 · 총 1.6M~2.5M 토큰 · 프로젝트 총괄 단독 수행보다 비용은 증가할 수 있습니다.
```

위 예시의 최상위 실행 주체는 추천 설정으로 고정된 프로젝트 총괄 에이전트입니다. 총괄은 리서치·검증·통합을 위임한 뒤 최종 결과를 통합하고 검증합니다. 독립 작업이 없다면 같은 총괄이 하위 에이전트 없이 프로젝트를 직접 수행합니다.

수치는 보장이 아니라 범위 예측입니다. 입력 자료의 양·품질, 도구/테스트 시간, 재시도, 검토 횟수, 병렬 가능 여부를 가정으로 밝힙니다. 토큰은 프로젝트 총괄, 하위 에이전트(사용 시), 검증·통합 작업을 합산합니다. 병렬 구성은 **완료 시간은 줄일 수 있지만 총 토큰과 비용은 늘 수 있습니다.**

비용은 실행 시점에 검증한 공식 단가 또는 사용자가 제공한 단가가 있을 때만 통화 범위로 표시합니다. 단가가 없으면 토큰 범위와 아래 산식만 제공합니다.

`예상 비용 = 입력 토큰 × 입력 단가 + 출력 토큰 × 출력 단가 + 도구/제공자 부과금`

## 반복 오류 기반 설정 조정

초기에는 기준 추천안만 제시한 뒤, 아래 중 하나를 **한 번만** 선택합니다.

- **고정**: 추천 모델·추론강도를 프로젝트 내내 유지합니다.
- **첫 조정 시 확인**: 최초 조정 신호에서만 제안 설정을 승인할지 묻습니다. 승인하면 이후 유사 조정은 자동 적용하고, 거절하면 프로젝트 설정을 고정합니다.
- **자동 조정**: 근거가 충분한 반복 오류·재작업 신호가 생기면 다음 작업 단위, 교체할 프로젝트 총괄, 또는 새 하위 에이전트의 설정을 자동 조정합니다.

조정 평가는 사용자가 요청했거나 아래 신호가 발생했을 때만 수행합니다. 단발성 오류나 일시적인 외부 도구 장애는 신호로 처리하지 않습니다.

- 같은 인수 테스트가 두 번 실패
- 수정 후 같은 오류가 재발
- 같은 인수 기준에서 재작업이 두 번 발생
- 프로젝트 총괄이 운영하는 하위 에이전트 결과가 인수 기준을 충족하지 못함

평가 시 아래 항목을 기준안과 비교합니다.

- 테스트·검증 결과, 구체적 오류, 사용자 피드백
- 재작업 횟수와 반복 실패
- 실제 경과 시간 및 확인 가능한 토큰 사용량
- 남은 작업의 난이도·실패 비용·독립적 병렬 가능성

조정안은 기존/제안 모델·추론강도, 평가 근거, 품질·시간·토큰·비용 영향 예측, 적용 범위를 함께 보여 줍니다. 일반적으로 추론강도부터 한 단계 조정하고, 여전히 능력 차이가 명확할 때만 모델 티어를 바꿉니다. 품질 문제가 있으면 상향, 품질이 충분한 비용·시간 문제면 하향을 우선 검토합니다.

일반적으로 추론강도부터 한 단계 조정하고, 여전히 능력 차이가 명확할 때만 모델 티어를 바꿉니다. 조정 후에는 다음 검증 결과가 나올 때까지 추가 조정을 하지 않습니다. 실행 중인 루트 에이전트의 모델은 바꾸지 않으며, 지원되는 Codex 환경에서 다음 작업 단위, 교체할 프로젝트 총괄, 또는 새 하위 에이전트에만 적용합니다.

|---|---|---|
| `none` | 없음 — 선택한 모델의 표준 단가 적용 | 추론 토큰을 사용하지 않는 지연시간 민감 작업에 적합 |
| `low` | 없음 — 선택한 모델의 표준 단가 적용 | 일반적으로 토큰·지연시간을 낮추는 방향 |

| `high` | 없음 — 선택한 모델의 표준 단가 적용 | 더 많은 추론 토큰이 발생할 수 있어 출력 과금이 증가할 수 있음 |

| `max` | 없음 — 선택한 모델의 표준 단가 적용 | 어려운 문제에만 사용; 실제 출력·추론 토큰 사용량을 확인 |

`ultra`는 이 API 토큰 단가표의 추론강도가 아닙니다. OpenAI 안내상 Codex/ChatGPT Work의 제품 설정으로 제공될 수 있으므로, API 비용 산정에는 위 모델별 토큰 단가를 사용하고 실제 Codex 구독·사용 한도는 해당 플랜을 확인해야 합니다. [GPT-5.6 출시 안내](https://openai.com/index/gpt-5-6/)

참고: 입력이 272K 토큰을 초과하면 전체 요청에 입력 2배·출력 1.5배 단가가 적용됩니다. 캐시 쓰기는 일반 입력 단가의 1.25배이며, 웹 검색·컴퓨터 사용 등 도구 호출, 이미지 입력, 우선 처리 등은 별도 과금될 수 있습니다. 최신 조건은 각 공식 모델 페이지에서 다시 확인하세요: [Sol](https://developers.openai.com/api/docs/models/gpt-5.6-sol), [Terra](https://developers.openai.com/api/docs/models/gpt-5.6-terra), [Luna](https://developers.openai.com/api/docs/models/gpt-5.6-luna).

## 빠른 사용

설치한 뒤 새 대화에서 다음처럼 요청하세요.

```text
이 프로젝트에 가장 적합한 모델과 추론강도를 추천해줘.
프로젝트: 여러 데이터 소스에서 경쟁사 리서치를 수집하고 보고서를 작성한다.
```

정보가 부족하면 스킬이 핵심 질문만 묻습니다. 바로 시작하려면 “추천한 설정으로 진행해줘”라고 말하세요. 그러면 프로젝트에 `gpt56-router-<tier>-<effort>.toml` 설정 파일을 만들고, 지원되는 경우 동일한 모델·추론 강도의 프로젝트 총괄 에이전트에게 전체 작업을 위임합니다.

## Codex 자동 적용 방식

이 스킬은 현재 대화 루트의 모델을 바꾸지 않습니다. 대신 사용자가 시작을 승인한 프로젝트에 아래와 같은 **프로젝트 범위**의 사용자 정의 에이전트 설정을 생성하고, 새 프로젝트 총괄 에이전트에 적용합니다.

```text
.codex/agents/gpt56-router-sol-high.toml
```

설정 파일에는 `model`과 `model_reasoning_effort`가 고정됩니다. 루트 대화는 사용자와의 의사결정·상태 공유를 맡고, 총괄 에이전트는 전체 작업·통합·검증을 책임집니다. 총괄은 다음 두 방식 중 하나로 실행합니다.

```text
루트 대화
  └─ 프로젝트 총괄 에이전트 (권장 모델·추론강도)
       ├─ 직접 실행: 서로 강하게 연결된 작업
       └─ 위임 실행: 독립 작업만 하위 에이전트에 분배 → 총괄이 통합·검증
```

따라서 하위 에이전트가 필요 없는 프로젝트도 동일하게 총괄 에이전트를 거칩니다. 같은 이름의 스킬 생성 파일은 다음 추천에서 갱신할 수 있지만, 사용자가 만든 에이전트 파일은 덮어쓰지 않습니다. 전역 `~/.codex/agents/` 설정은 사용자가 별도로 요청한 경우에만 변경합니다.

## 설치

먼저 저장소를 내려받습니다.

```bash
git clone https://github.com/seominpapa/gpt56-model-router.git
cd gpt56-model-router
```

### Codex

개인 스킬 폴더에 복사하면 모든 프로젝트에서 사용할 수 있습니다.

```bash
mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
cp -R . "${CODEX_HOME:-$HOME/.codex}/skills/gpt56-model-router"
```

Codex를 새로 시작한 뒤 `$gpt56-model-router`를 호출하거나, 모델 선택 요청을 자연어로 입력하세요.

### Claude Code

Claude Code는 Agent Skills의 `SKILL.md` 형식을 지원합니다. 개인 스킬로 설치하려면 다음을 실행하세요.

```bash
mkdir -p ~/.claude/skills
cp -R . ~/.claude/skills/gpt56-model-router
```

특정 프로젝트에서만 쓰려면 해당 프로젝트 루트의 `.claude/skills/` 아래에 복사하세요.

```bash
mkdir -p .claude/skills
cp -R /path/to/gpt56-model-router .claude/skills/gpt56-model-router
```

`/gpt56-model-router`로 호출하거나 프로젝트 시작 시 모델·추론 강도 추천을 요청할 수 있습니다.

### Cursor

프로젝트 루트에 다음 경로로 배치하세요.

```bash
mkdir -p .cursor/skills
cp -R /path/to/gpt56-model-router .cursor/skills/gpt56-model-router
```

Cursor Agent를 다시 열고 `/gpt56-model-router`를 호출하거나, 새 프로젝트의 모델·추론 강도 추천을 요청하세요.

## 플랫폼별 적용 범위

이 스킬의 추천 로직은 세 도구에서 공통으로 사용할 수 있지만, 실제 모델 변경은 도구와 계정의 모델 제공 범위에 따라 다릅니다.

| 도구 | 추천 | GPT-5.6 설정 적용 |
|---|---|---|
| Codex | 지원 | 시작 승인 후 프로젝트 총괄 설정 파일을 생성하고, 현재 환경에서 지원되는 모델·추론 설정으로 총괄 에이전트에 자동 적용 |
| Claude Code | 지원 | 추천·작업 설계 용도로 사용. GPT-5.6을 Claude 하위 에이전트에 자동 적용할 수 없음 |
| Cursor | 지원 | 추천 용도로 사용. Cursor에 표시된 모델은 수동 선택 또는 CLI 지정 가능하지만, 이 스킬이 하위 에이전트에 자동 고정할 수 없음 |

### 중요한 제한

Codex만 이 스킬이 프로젝트용 에이전트 설정 파일을 만들고, 지원되는 모델·추론 강도로 **새 프로젝트 총괄 에이전트에 자동 적용**할 수 있습니다. 총괄이 하위 에이전트를 만들 수 있는지는 현재 Codex 환경의 지원 범위에 따릅니다.

- **Claude Code**: 하위 에이전트는 만들 수 있지만 Claude 모델만 사용합니다. OpenAI GPT-5.6을 하위 에이전트의 모델 프로바이더로 자동 적용할 수 없습니다.
- **Cursor**: GPT 모델을 선택할 수 있는 계정이라도, `SKILL.md`가 Cursor의 하위·백그라운드 에이전트 모델을 직접 변경하거나 고정하는 공식 메커니즘은 없습니다. 모델은 Cursor UI, Custom Mode 또는 CLI에서 사용자가 선택해야 합니다.

스킬은 지원되지 않는 설정을 적용했다고 말하지 않습니다. 대신 선택해야 할 모델과 추론 강도를 명확히 알려줍니다.
